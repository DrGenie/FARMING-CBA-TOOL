Technical appendix

This appendix describes how the tool reads data, constructs yield and cost deltas and calculates economic indicators. It is written for analysts who want to understand or reproduce the methods.

Data model

The tool expects plot level data with a treatment label, a replicate identifier, a plot identifier, a yield measure and one or more cost variables. Where a data dictionary is available it is used to match roles. Otherwise the tool infers roles from column names.

Treatments are grouped using the treatment label. Within each replicate the tool identifies control plots and calculates the mean control yield and cost per hectare. Each treated plot is then compared with the control mean for the same replicate.

Yield deltas are computed as yield on the treated plot minus the mean yield of control plots in the same replicate. Cost deltas are computed in the same way using costs per hectare. This approach reduces the influence of level differences between replicates.

From plot level data to treatment summaries

For each treatment the tool aggregates plot level deltas to obtain mean effects. It records the number of yield observations, the mean yield, the standard deviation of yield, the mean yield delta relative to the control and the standard deviation of this delta.

The tool also summarises cost deltas. Labour, services and materials costs are separated using column name patterns. Within each group the mean incremental cost per hectare is recorded for each treatment relative to the control.

Outliers in yield deltas are flagged using the interquartile range rule. Treatments with very few observations are flagged so that users can treat their results with caution.

Cashflow construction

For each treatment the tool builds a stream of annual benefits and costs. Benefits are based on the yield delta per hectare multiplied by grain price, farm area, adoption and a risk adjustment. Yield effects can be limited to a fixed number of years through the persistence setting.

Costs are split into capital and recurring components. Capital costs occur in year zero. Recurring costs are built from labour, services and materials cost per hectare multiplied by the treated area. A recurrence setting allows these application costs to repeat every given number of years or only in year zero.

For any given treatment the tool then holds a vector of net cashflows equal to benefits minus costs for each year in the analysis horizon.

Economic indicators

The present value of benefits is the sum of discounted annual benefits. The present value of costs is the sum of discounted annual costs. Net present value is the difference between the present value of benefits and the present value of costs.

The benefit cost ratio is the present value of benefits divided by the present value of costs. The return on investment is net present value divided by the present value of costs, expressed in percent. These indicators are computed for each treatment and then compared with the control.

The internal rate of return is the discount rate that produces a net present value of zero for the stream of net cashflows. The modified internal rate of return uses separate finance and reinvestment rates and is reported where meaningful. The payback period is the first year in which the cumulative discounted cashflow becomes non negative.

Sensitivity analysis

Sensitivity analysis is performed by varying four factors. These are grain price, discount rate, persistence of yield effects and recurrence of costs. The tool builds a grid of scenarios across these four dimensions and recomputes net present value, benefit cost ratio and return on investment for each treatment and scenario.

The sensitivity summary page reports how wide the scenario grid is and highlights extremes. Analysts can export the full grid to a CSV file for further work in a spreadsheet or statistical package.

Data checks

During import the tool runs a set of checks. These include detection of missing key variables, absence of control observations, negative yields, non numeric cost entries and low replication for treatments. Each check is tagged with a severity level so that users can focus on problems that would make economic results unreliable.

Error level checks prevent the dataset from being committed. Warning level checks do not block analysis but are highlighted on the Data tab and in the AI briefing prompt.
